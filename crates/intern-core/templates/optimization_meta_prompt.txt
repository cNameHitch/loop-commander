## Your Objective

Analyze the execution history above and produce an optimized version of the current
command. Your optimization must demonstrably address at least one observed pattern
from the execution history. Do not optimize for hypothetical problems not evidenced
in the data.

### Analysis Criteria

Examine the history for the following signal types, in priority order:

1. FAILURE PATTERNS — Repeated non-zero exit codes with identical or similar stderr.
   These indicate structural prompt flaws (ambiguous instructions, missing context,
   incorrect assumptions about environment).

2. TOKEN WASTE — High cost_usd or long duration_secs on runs that produced
   minimal or low-quality output. Look for verbose reasoning that did not improve
   outcomes, redundant re-checking behavior, or open-ended instructions that cause
   Claude to over-generate.

3. OUTPUT INCONSISTENCY — Runs with identical inputs but structurally different
   stdout (different formats, different section headings, variable levels of detail).
   This indicates underspecified output format instructions.

4. TIMEOUT PROXIMITY — Runs where duration_secs is within 20% of any apparent
   ceiling (e.g., multiple runs near 55-60s suggests a 60s timeout risk). Indicate
   scope reduction or early-exit strategies.

5. EDGE CASE BLINDNESS — Stderr warnings or stdout content indicating the task
   encountered conditions it was not instructed to handle (empty inputs, missing
   files, API rate limits, etc.).

6. SCOPE MISMATCH — Output that is consistently far more or far less detailed than
   the task description implies is needed.

### Optimization Constraints

- Preserve the intent and purpose of the original command exactly.
- Do not add capabilities the user did not request.
- Do not remove capabilities unless the execution history shows they are causing
  failures or excessive cost.
- If the current command is already well-optimized and the history shows consistent
  success, say so explicitly and return the command unchanged with
  confidence_score: 100 and a changes_summary stating no changes were needed.
- Optimized commands must remain valid as claude -p arguments.
